因为KINDLE对PDF扫描文档的渲染、对比度、读取等都有先天不足，就算用上多看系统，效果也是不理想，所以用KINDLE看PDF绝对是一种折磨。在原版系统下，我们如何体验工具在KINDLE的魅力呢。本人参考网上所说和自己实践得出相对好的方法。理论上就是将PD转换为MOBI文件，这样的PDF所存在的问题大部份就解决了，不会出现看PDF文件时的内存不足。下面为“汇编语言（王爽）.PDF”为例，具体操作如下：
1、先用工具A将PDF文件切边，因为我要转成MOBI文件，留上下左右的空白边完全没完成没用，还阻碣阅读。所以第一步先成PDF空白地方切掉。
2、PDF是连续文档，可看成条状，用B工具将切边后的文档，剪成分辨率为758X1024连序的图片，按成我的实践多次效果，基本用这个工具剪，不会剪到文字。具体效效如下所示
3、用工具C将有序的图像文件生成MOBI文件。这样MOBI版的PDF就出来。
最后我拿一本500页的16开PDF的文档测试，转换成MOBi的全过程总共15分种 ，因为这三个软件最是一次性操作，无须重复，只是有些等待。至于效果，转成MOBI基本上没有卡死过，因为本质变成纯图像文件，Kindle的渲染很完美，基本不存在虚光、对比度低下、发散等现象，无需缩放阅读效果和文字版MOBI的差不多（原PDF文档为普清以上）。缺点的就是转换后的文件较大，原16开的350页左右，转成MOB后约180MB，但打开速度和翻页速度和普通MOBI无大区别（LCHJZJF）。

教程：
一、下载这三个工具：A、PDF切边工具briss；B、PDF分割成图片pdflrf ；C、mobi制作软件ChainLP；顺便准备好PDF文件。
二、打开Briss文件夹，运行briss-0.9.exe-----File-----load file 打开PDF文件，在输入框 里输入1-30，因为PDF文件都有封面和目录之类的，那些最好是独立页面切边，所以建议输入第1至30页。最后根据自己的习惯调选框来切边。确定后，点击Action---------Crop PDF保存切好边的PDF。
三、 打开Pdflrf文件夹，运行pdflrf900.exe--------Input选切好边的PDF，然后将Output的输出文件的后缀格式名改为ZIP和将Width设为758，Height设为1024，设好后点击Convert输出剪好的图片集。
四、打开ChainLP文件夹，运行ChainLP.exe将剪好的图片文件夹拖进ChainLP的左窗，其他参数最好不要动，设好输出文件夹，点击Output，就生成MOBI了
