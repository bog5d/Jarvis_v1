# python 相关概念：计算机

* 概念：计算机

1. 能够根据一组指令操作数据的机器
2. 两大特性：

A功能性：可以进行数据运算。
B 可编程性：根据一系列指令来执行。

[概念本.docx](./_resources/python_相关概念：计算机.resources/概念本.docx)

* 概念：存储程序。

“存储程序”原理，是将根据特定问题编写的程序存放在计算机[存储器](http://baike.baidu.com/view/87697.htm%20/t%20_blank)中，然后按存储器中的存储程序的首地址执行程序的第一条[指令](http://baike.baidu.com/view/178461.htm%20/t%20_blank)，以后就按照该程序的规定[顺序执行](http://baike.baidu.com/view/2120589.htm%20/t%20_blank)其他指令，直至程序结束执行。
将编写好的程序，放在计算机存储器中，然后按照存储器中的存储程序的首地址执行程序的第一条指令，以后就按照该程序规定顺序执行其他指令，直至程序结束执行。

* 程序设计语言种类

1. 机器语言：01代码。CPU认识的语言

例如：2+3的运算 1101001000111011

2. 汇编语言：在机器语言上增加了人类可读的助记符

例如：2+3的运算 add 2,3,result

3. 高级语言：向自然语言靠近的语言

例如：2+3的运算result=2+3

* 编译和解释

1. 编译：将高级语言源代码转换成目标代码（机器语言），程序便可以运行。
2. 解释：将高级语言源代码逐条转换成目标代码同时逐条执行，每次运行需要源代码和解释器。

编译是将源程序翻译成可执行的目标代码，翻译与执行是分开的；而解释是对源程序的翻译与执行一次性完成，不生成可存储的目标代码。这只是表象，二者背后的最大区别是：对解释执行而言，程序运行时的控制权在解释器而不在用户程序；对编译执行而言，运行时的控制权在用户程序。
编译就相当于直接翻译成A机器语言，A直接执行。 但B机器要执行同样的指令，就需要重新编译成适应B的。
解释就是把指令翻译成中间语言，就像我们出国都用通用的英语，我说英语比较吃力，同样越南人听我说英语，理解起来也比较慢，但是全球都可以用。
编译就相当于我学外语学的越南语，去越南很顺利。 但去日本就用不上，还得再学日语。

解释就相当于，我学英语，去越南，日本，都能用，但是都不太顺畅，要慢慢去理解查字典，很消耗大脑资源，但是通用啊。
